Configuration Guide
How to Configure Workflow Levels:
  1. Level Order: Sequential number (1, 2, 3, ...). Determines the approval sequence.
  2. Level Name: Descriptive name shown to users.
  3. Role: User group that can approve at this level (e.g., ROLE_LEVEL1, ROLE_MANAGER).
  4. Next Level: The level order to route to after approval. Leave empty for the final level.
  5. Query Return Level: Default level to send queries to (usually Level 1).
  6. Enabled: Uncheck to temporarily disable a level without deleting it.
Example Configurations:
Simple 2-Level Workflow: 
Level 1 → Level 2 → End
next_level: 2    null
Complex 4-Level Workflow: 
Level 1 → Level 2 → Level 3 → Level 4 → End
next_level: 2    3         4         null
